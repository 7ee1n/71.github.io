iot-protocol-overview

Posted on 2021-07-29 Edited on 2021-08-23
Symbols count in article: 2.3k Reading time ≈ 2 mins.

Seebug入门级资料 的note。简单来说攻击面可以分成设备本身、移动端、云端、通信过程四部分。以下是整理的通讯层面的内容，个人感觉重点是 协议的实现和数据格式。

通讯层面-通用协议标准

以下的这些协议都会涉及到应用层，有一部分还会规定传输层、数链层、物理层的标准。

CoAP (Constrained Application Protocol)

iot设备的 RAM 和 ROM 一般很小，没有足够的空间支持 TCP 和 HTTP 协议，也就诞生了这个由 UDP 实现的低功耗短连接的协议。主要用于 M2M 的应用场景，也就是能联网的设备，像水表的自动抄表模块、手机的移动通信模块、工业的遥控模块等。（工具）

报文格式

可以理解为简化的HTTP（text），分请求和响应报文，但 CoAP 是 binary 格式的。除去token、options、payload，最小的报文长度为 4 字节。各固定字段含义和内容详见 wiki。其中Type大小为 2 bit，共四种类型：CON、NON、ACK、RST，前两种可类比为 TCP 和 UDP来理解。

URL

默认端口为 udp 5683（coaps是5684）。

MQTT (Message Queuing Telemetry Transport)

也是轻量级的通讯协议，但它基于的是 TCP/IP 协议。CoAP支持的是 Client 和 Server 间的一对一信息传递，MQTT支持多对多，应用场景更广，如卫星链路通信传感器、智能家居等。

报文类型和格式

主要分 Connect、Subscribe、Publish、Disconnect四种类型。客户端可以是发布者或订阅者，服务器端是代理者。下图以订阅过程为例，或者参考 wiki 上的智能空调温度调节实例，个人觉得那个更好理解。

报文的最小长度为 2 字节，payload最长为256字节。固定头部和可变长头部包括消息类型、长度等常见字段，详见 菜鸟教程 。

AMQP (Advanced Message Queuing Protocol)

跟MQTT一样都是队列协议，都使用异步通信机制。可以将 MQTT 理解为 AMQP 的简化版，它们的实现原理相同，但 MQTT 是专门为计算性能不高的小设备设计的。两者主要区别如下表，详见：RabbitMQ MQTT协议和AMQP协议 。

AMQP MQTT
底层消息传递协议 消息通讯机制
面向缓存 面向流，内存占用低
支持长周期消息传递 短周期，不支持长周期存储和转发
用SASL和TLS确保安全性 简单用户名和密码
Client和Server对等 Client和Server不对等
可扩展 不可扩展

CANbus（Controller Area Network）

多主机串行标准，为实时要求而设计，在微控制器之间提供快速通信，取替了成本更高的RAM技术。主要用于实时性要求高的嵌入式系统，如汽车、工业自动化、医疗工程等。更具体的可以看 A Brief Introduction to Controller Area Network 。

架构

需要高低电平两条电线：CAN_H 和 CAN_L，组成一条双绞线。

具体看 wiki ，感觉跟通信原理和计组很像，主要是底层的一些编码跟电路的关系。

Modbus

多对多的请求应答协议，用于控制器和设备通信，主要是规定了数据传输的模式、格式，带有检错功能。

校验过程：Modbus通讯协议学习 。

Profibus

前面的Modbus是施耐德电气公司发明的。Profibus则是西门子公司发明的，实时性高，用于智能设备、自动化系统间的点对点通信。更详细的物理层上的结构可以看 ：What is PROFIBUS? | SMAR Technology Company

DNP3（Distributed Network Protocol 3）

分布式网络协议，比 Modbus 更复杂、更高效。为用户数据提供复用、数据分片、错误检查、链路控制、优先级和数链层寻址服务，定义了传输功能，使用大量的 CRC 校验来检错。

详见：DNP3协议简单介绍及协议识别方法 – 灯塔实验室 。

通信端口：TCP的20000端口。

BACNet（Building Automation and Control）

building 指的是楼宇，该协议主要用于供暖、空调控制、火灾探测系统等。其协议栈参考：认识BACnet协议 - 薄心之心 - 博客园 。

HL7（Health Level 7）

数据格式看 wiki 的就够了。

XMPP（Extensible Messaging and Presence Protocol）

基于 XML 类型，为即时消息、在线状态信息而设计，实时性高。

代码详见： XMPP详解 - 简书 。

UPnP（Universal Plug and Play）

即插即用协议，我一开始不是很理解这个 “插” 字。顾名思义，Plug 🔌，就是插上🔌后联网设备能立即连网，也就是建立功能性网络服务。因此它会使用 HTTP 协议进行设备搜索请求（UDP 1900端口）、数据传输。有的设备还能通过网络访问，实际上是 UPnP 在刚开始查找到设备后，有可能为其增加一个外部 IP 地址，修改路由器的内容，增加新的端口转发映射，使得打印机、相机等设备能通过网络访问。
